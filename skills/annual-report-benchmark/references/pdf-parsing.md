# 年报解析标准（单一最佳方案）

## 1. 唯一解析路径

- 只允许：`PDF -> DOCX`（使用 `pdf2docx-mcp`）
- 从 DOCX 文件中提取文本和表格数据进行字段识别
- PDF 仅可用于转换与页数校验，不可用于字段提取

## 2. 执行要求

- 每份 PDF 仅转换一次
- 后续取数仅基于生成的 DOCX 文件
- 进入取数阶段后，所有输入文件扩展名必须为 `.docx`

## 3. 禁止操作（硬性）

- 禁止 PDF to Text
- 禁止 OCR PDF
- 禁止直接读取 PDF 页面文本/表格做字段提取
- 若发生上述任一操作，当前轮结果作废，必须回到 DOCX 重新提取

## 4. MCP 工具使用

使用 `pdf2docx-mcp` 提供的转换工具：

- `mcp__plugin_pdf2docx-mcp_pdf2docx-mcp__convert` - 执行 PDF 到 DOCX 转换
- `mcp__plugin_pdf2docx-mcp_pdf2docx-mcp__get_info` - 获取 PDF 元信息（页数完整性验证）

## 5. 最低验收

DOCX 中必须同时满足：

**章节验收**（至少命中以下 1 个章节标题）：

- `主要会计数据和财务指标`
- `主要财务数据`

**核心字段验收**（至少命中以下 2 个字段名）：

- `营业收入`
- `归属于母公司股东的净利润`
- `经营活动产生的现金流量净额`

> 注：`合同负债` 等报表基础项字段不作为最低验收条件——部分公司无合同负债，不应因此判定转换失败。

## 6. 失败处理

- 若最低验收未通过，当前 PDF 判定失败
- 进入缺失占位 + 原因说明流程，不得再次重试

## 0. 术语

- 报告期：`Y`
- 对比期：`Y-1`
- 输出字段名：仅允许使用 `references/required-fields.md` 中的 23 项标准名称。

## 1. 输入与下载

- 每家公司必须下载 `Y` 与 `Y-1` 两份年报。
- 单家公司失败不阻断其他公司——前提是已产出该公司的降级表（可提取字段 + 缺失清单 + 失败原因）；降级表视为已产出，满足"产出单公司表"要求后方可继续处理下一家。

## 2. 解析标准

- 解析标准已在 `references/pdf-parsing.md` 中完整定义（开工前已按顺序加载，此处不再重复读取）。
- 每份 PDF 仅执行一次 `PDF -> DOCX` 转换（使用 `pdf2docx-mcp`）。
- 后续取数仅基于该 DOCX 文件，不再回退其它格式。
- 字段提取阶段禁止 PDF to Text / OCR PDF / 直接解析 PDF 页面文本。
- 若误用 PDF 直读，当前轮结果必须作废并基于 DOCX 重做。

## 3. 单公司执行顺序

1. 先按 23 项字段构建空骨架。
   - 执行前自检：本步骤及后续步骤输入文件必须全部为 `.docx`。
2. 提取核心章节：主要会计数据和财务指标。
3. 提取报表基础项：资产负债表、利润表、现金流量表。
4. 提取员工薪酬章节。
   - **应付职工薪酬本期减少合计金额**：位于附注"应付职工薪酬列示"（或"职工薪酬"、"职工薪酬汇总"等类似表格），取合计行的"本期减少"（或"本期支付"、"本期转出"等同义列）对应的数值。
   - 该表通常含短期薪酬、离职后福利、辞退福利等分项，取最底部"合计"行（部分公司称"小计"）在"减少"方向的数值。
   - 注意：不同公司的列标题可能略有差异，识别时以"减少方向 + 合计行"为定位原则，不拘泥于精确列名。
5. 搜索"尚未履行或尚未履行完毕的履约义务所对应的收入金额"相关描述：
   - 如有披露数值，则提取并加入当前公司表，参与对标。
   - 如未披露（含"无"、"不适用"、空白），则跳过，不占位，不影响覆盖率校验。
6. 计算派生指标（按 `metric-definitions.md`）。
7. 回填对比期值：
   - 优先取 `Y` 年报中的对比期列（通常为"上年同期"或"上年末"列）。
   - 若 `Y` 年报对比列缺失，则从 `Y-1` 年报补提，并将数据质量标记为"跨年报补齐"。
   - 若两者均有数值但不一致（如追溯调整）：以 `Y` 年报对比列为准（反映公司最新口径），在备注中注明"与 Y-1 年报存在差异，采用 Y 年报披露口径，差异原因可能为追溯调整或会计政策变更"。
8. 计算同比并写入质量标记。
9. 执行 23 项覆盖率校验；缺失项必须占位。

## 4. 两轮止损（硬性）

- 每家公司最多 2 轮自动提取。
- 第 1 轮：完整流程提取。
- 第 2 轮：仅允许针对缺失项做定向补提。
- 第 2 轮后仍不达标：立即降级，产出
  - 单公司对比表（可提取字段）
  - 缺失项清单
  - 失败原因说明
- 未产出当前公司单公司表，不得进入下一家公司或总表聚合。

## 5. 同义词识别映射（仅识别，不可作为输出名）

- `归属于上市公司股东的净利润` -> `归属于母公司股东的净利润`
- `在职员工的数量合计` -> `报告期末在职员工的数量合计`

## 6. 可追溯字段（每行至少包含）

- 指标名称
- `Y年值`
- `Y-1年值`
- 同比
- 单位
- 数据质量
- 来源章节
- 报告期值来源年报
- 对比期值来源年报
- 备注

## 7. 多公司聚合

- 必须先完成全部单公司表，再生成总对比大表。
- 聚合时不得删除缺失指标行。
